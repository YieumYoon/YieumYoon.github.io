Astro Blog Architecture

A read-only architecture guide for the existing hosted Astro blog in this repository. This document describes the current structure only; it does not propose code changes as facts.

1. High-Level Overview

This project is a simplified static personal blog generated with Astro. It appears to be based on the astro-sphere template and is customized with Korean blog content, RSS, sitemap generation, Google Analytics, and GitHub Pages deployment.

  • Main framework: astro version range ^4.4.13.
  • Package manager: npm, based on package-lock.json.
  • UI/runtime integrations: Tailwind, RSS, sitemap, Partytown.
  • Local Node hint: .nvmrc contains v22.12.0.

Important Commands

Command Defined as Purpose
npm run dev astro dev Start local Astro dev server.
npm run dev:network astro dev --host Start dev server available on the network.
npm run build astro check && astro build Type/content check, then static production build.
npm run preview astro preview Preview the built site locally.
npm run lint eslint . Run ESLint over the project.

5. Post Rendering Flow

Example real post: src/content/blog/00001-started-astro-github-blog/index.md. Its content collection slug becomes 00001-started-astro-github-blog, and the generated URL is /blog/00001-started-astro-github-blog/. 

src/content/blog/00001-started-astro-github-blog/index.md
  -> registered as part of the "blog" content collection
  -> src/pages/blog/[...slug].astro getStaticPaths()
  -> Astro.props is the selected post entry
  -> PageLayout sets HTML shell, head, header, drawer, footer
  -> TopLayout contains ArticleTopLayout
  -> BottomLayout contains ArticleBottomLayout
  -> ArticleBottomLayout calls await entry.render() and outputs <Content />
  -> global.css and Tailwind classes style the rendered article

8. Styling

Styling is a mix of global CSS, Tailwind utility classes, Tailwind's typography plugin, and a small amount of component-local <style> or inline scripting.

  • Global CSS: src/styles/global.css.
  • Tailwind config: tailwind.config.mjs.
  • Global CSS import location: src/layouts/PageLayout.astro.
  • Theme script: public/js/theme.js.
  • Other global scripts: public/js/scroll.js, public/js/animate.js, public/js/bg.js.

Colors

Colors are mostly Tailwind utility classes directly in Astro markup, such as bg-white, dark:bg-black, text-black/75, and dark:text-white/75. No separate color token file was found.

Typography

  • src/styles/global.css defines the Atkinson font faces.
  • tailwind.config.mjs extends Tailwind's sans font family with Atkinson.
  • article uses Tailwind Typography via prose dark:prose-invert.
tailwind.config.mjs
  -> scans src/**/*.{astro,html,js,md,ts}
  -> extends font, typography, rotations, animations

src/layouts/PageLayout.astro
  -> imports src/styles/global.css
  -> every route using PageLayout receives global styles

public/js/theme.js
  -> toggles html.dark
  -> Tailwind dark: classes change theme colors

9. Deployment

The site is deployed to GitHub Pages through .github/workflows/astro.yml.

  1. Workflow runs on pushes to main and manual dispatch.
  2. It checks out the repository.
  3. It detects npm from package.json/package-lock.json.
  4. It sets up Node 20 in GitHub Actions.
  5. It runs npm ci.
  6. It runs npx --no-install astro build --site ... --base ....
  7. It uploads dist as a Pages artifact.
  8. It deploys using actions/deploy-pages@v4.

Astro's configured site URL is https://yieumyoon.github.io in astro.config.mjs. The workflow also passes GitHub Pages' detected origin and base_path into astro build.

10. Things To Understand Before Editing

Potential Risks

  • Local Node version is v22.12.0, while GitHub Actions uses Node 20.
  • public/rss.xml and public/sitemap-*.xml coexist with generated RSS/sitemap routes. It is unclear whether these public files are stale, intentional, or template leftovers.
  • Site-wide navigation and labels are centralized in src/consts.ts; changing paths there affects header and drawer navigation.
  • Global layout includes analytics and scripts; edits to PageLayout or BaseHead affect every page.

Safe Areas To Customize First

  • Add or edit blog posts under src/content/blog/, while preserving the required frontmatter schema.
  • Update site text, navigation labels, and social links in src/consts.ts.
  • Make small homepage content edits in src/pages/index.astro.
  • Adjust list-card text presentation in ArrowCard.astro after understanding that it is shared by the homepage and blog list.
  • Tune global typography in src/styles/global.css or tailwind.config.mjs, remembering that this affects the whole site.

Rule of thumb: content files are the lowest-risk place to start. Layouts and global head/style files have the widest impact.
